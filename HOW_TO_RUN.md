# 🏋️ دليل تشغيل وتطوير موقع Master Gym (المشروع المقسّم)

تم تقسيم المشروع إلى مجلدين مستقلين ليسهل رفعهما وتشغيلهما بشكل منفصل ومحمي:
1. **`master-gym-admin`**: يحتوي على لوحة تحكم الإدارة (الفرونت إند) بالإضافة إلى خادم الـ API (الباك إند).
2. **`master-gym-player`**: يحتوي على واجهة اللاعبين (الفرونت إند الاستاتيك).

---

## 💻 التشغيل المحلي (Local Development)

لقد قمنا بإعداد المشروع بحيث يمكنك تشغيله **مباشرة من المجلد الرئيسي** (دون الحاجة للدخول إلى المجلدات الفرعية)، كالتالي:

### 1️⃣ ثبّت المكتبات (مرة واحدة فقط)
افتح سطر الأوامر (PowerShell أو Terminal) في المجلد الرئيسي للمشروع واكتب:
```powershell
npm install
```
> هذا سيقوم تلقائياً بتثبيت كل المكتبات والاعتمادات اللازمة داخل المجلد الفرعي.

### 2️⃣ تأكد من ملف `.env`
تأكد من وجود ملف `.env` داخل المجلد الفرعي `master-gym-admin/` ويحتوي على بيانات الاتصال بـ Supabase و Gemini:
```env
SUPABASE_URL=https://xxxxxx.supabase.co
SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
GEMINI_API_KEY=AIzaSy...
```

### 3️⃣ شغّل السيرفر
في المجلد الرئيسي للمشروع، اكتب الأمر التالي:
```powershell
npm start
```
سيعمل السيرفر ويقوم بتشغيل الباك إند واللوحة الإدارية على المنفذ المحلي: `http://localhost:3000`

---

## 🎮 تشغيل واجهة اللاعبين (Player Frontend)
لتشغيل واجهة اللاعبين محلياً، ما عليك سوى فتح ملف `master-gym-player/index.html` في المتصفح أو تشغيله عبر Live Server. ستقوم الواجهة بالاتصال تلقائياً بخادم الباك إند على المنفذ `3000` لمعالجة المحادثات.


## 🌐 روابط التشغيل المحلي السريعة

عند تشغيل السيرفر محلياً:

| الواجهة / الخدمة | الرابط | الوصف |
| :--- | :--- | :--- |
| **🎮 واجهة اللاعب** | افتح `master-gym-player/index.html` | واجهة المشتركين لمتابعة الأوزان والنقاط ومحادثة البوت |
| **👑 لوحة التحكم** | `http://localhost:3000` | لوحة المدير لإدارة المشتركين، الاشتراكات والتقارير |
| **🤖 بوت الأدمن** | `http://localhost:3000/bot.html` | مساعد الذكاء الاصطناعي الخاص بالمدير |
| **💓 فحص السيرفر** | `http://localhost:3000/health` | للتحقق من جهوزية اتصال Gemini والـ API |

## 🚀 الرفع والنشر السحابي (Deployment)

تم تهيئة المشروع بالكامل ليكون جاهزاً للنشر:

### 🅰️ نشر الباك إند + لوحة الأدمن (على Render)
1. قم بإنشاء خدمة ويب جديدة (Web Service) على موقع **Render**.
2. اربط حسابك بـ GitHub واختر هذا المستودع.
3. قم بتعيين المجلد الأساسي (Root Directory) ليكون: `master-gym-admin`.
4. اختر بيئة العمل لتكون `Node` وأمر التشغيل `npm start`.
5. أضف متغيرات البيئة (Environment Variables):
   * `SUPABASE_URL`
   * `SUPABASE_ANON_KEY`
   * `GEMINI_API_KEY`
6. سيقوم Render بنشر السيرفر وتوفير رابط URL عام له (مثال: `https://my-server.onrender.com`).

### 🅱️ نشر واجهة اللاعب (على Netlify)
1. قم بإنشاء موقع جديد على **Netlify**.
2. اربط المستودع وحدد المجلد الأساسي (Base Directory) ليكون: `master-gym-player`.
3. لا توجد حاجة لأوامر بناء (Build command فارغ)، ومجلد النشر هو المجلد الحالي `.`.
4. عند النشر، سيقوم Netlify بتوفير رابط عام لفرونت إند اللاعبين.
5. *(ملاحظة: تأكد من تحديث رابط السيرفر العام في ملفات الـ HTML عند الانتقال للإنتاج).*
